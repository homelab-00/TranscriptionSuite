## 0.6 Shared API facts (verified against the code ‚Äî trust these over stale docs)

- `GET /api/admin/status` (admin-gated) returns `{ status, version, models: model_manager.get_status(), config: {...} }`. Inside `models`: `job_tracker` (`{is_busy, active_user, active_job_id, cancellation_requested, progress: {current,total,message}|null, result}`) and `features` (`{diarization: {available, reason}, mlx: {...}, ...}`). The dashboard type `AdminStatus.models` is `Record<string, unknown>`, so existing call sites cast `(status.models as any)`.
- `AdminStatus.features` (top-level in `dashboard/src/api/types.ts`) is a RED HERRING ‚Äî the server never populates it there. `ServerStatus.diarization_available` is dead code. Use `models.features.*`.
- `useAdminStatus(pollInterval = 10_000)` (`dashboard/src/hooks/useAdminStatus.ts`) polls the endpoint via TanStack Query. `useServerStatus` polls the public `GET /api/status` every 10s.
- The startup/download event pipeline: server `emit_event()` (`server/backend/core/startup_events.py`) appends JSONL to `$STARTUP_EVENTS_FILE` (bind-mounted; host path created by `dockerManager.startContainer`) ‚Üí Electron `StartupEventWatcher` tails it ‚Üí IPC `activity:event` ‚Üí `useBootstrapDownloads.ts` ‚Üí Zustand `activityStore` (upsert by `id`) ‚Üí floating widget `dashboard/components/ui/ActivityNotifications.tsx`. Repeated `emit_event` calls with the SAME id update one card; a new id creates a new card.
- The single-slot job model: `TranscriptionJobTracker` (`server/backend/core/model_manager.py:54-183`) allows ONE active transcription job system-wide; `update_progress(current, total, message="")` is in-memory only (no DB column anywhere ‚Äî do not try to read progress from the durability table).

# Issue #209 ‚Äî PyAnnote diarization enabled by default when no HF token is configured (bug)

**Branch:** `fix/209-diarization-token-gate` (first of the Session-tab cluster ‚Äî do this before #212/#210/#211)

**Root cause (confirmed):** `dashboard/components/views/SessionImportTab.tsx:89` hardcodes `const [diarization, setDiarization] = useState(true);` with zero awareness of token/feature state. `dashboard/components/views/NotebookView.tsx:1549` has the identical bug. The server already computes exactly the needed signal ‚Äî `model_manager.get_status()['features']['diarization'] = {available: bool, reason: 'token_missing'|'ready'|...}` ‚Äî and it is already reachable in both components via the `useAdminStatus()` hook they already call. When a diarization-enabled job runs with no token, the server silently completes the job WITHOUT diarization and records `{requested: true, performed: false, reason: 'token_missing'}` in the job result ‚Äî which the dashboard captures (`importQueueStore.ts:444`) but never renders anywhere.

**Fix (3 parts, all dashboard):** (1) gate the toggle on `features.diarization.available` in both components; (2) force `enable_diarization: false` in the request payload when unavailable; (3) surface the "diarization was skipped" reason on completed jobs so silent skips become visible.

**No server changes.** The graceful-skip server behavior is correct (per the no-data-loss invariant) and already tested (`server/backend/tests/test_parallel_diarize.py::test_diarization_model_load_fails_graceful_fallback`).

**Caveat to preserve in copy:** the feature flag is computed ONCE at container startup (`ModelManager._initialize_diarization_feature_status`, never recomputed by `reload_config`). Adding a token in Settings requires a server restart to unlock. UI copy must say so.

**Files:**
- Modify: `dashboard/components/views/SessionImportTab.tsx` (toggle at ~89, `handleFiles` payload at ~296-340, `updateSessionConfig` effect at ~217-238, AppleSwitch render at ~771-780, `statusLabel` at ~446-464)
- Modify: `dashboard/components/views/NotebookView.tsx` (toggle state at ~1549, payload uses at ~1632/~1705, its own `statusLabel`)

# Issue #212 ‚Äî Make TXT/SRT output selection explicit beside Session file import (bug + UX)

**Branch:** `fix/212-output-format-selector` (Session-tab cluster ‚Äî branch AFTER the #209 PR is merged into the super-branch)

**Root cause (confirmed):** Output format for Session imports is decided 100% client-side in `dashboard/src/services/transcriptionFormatters.ts::resolveTranscriptionOutput` (lines 150-175), keyed off (a) the GLOBAL Settings toggle `output.hideTimestamps` (default false, lives in a different modal) and (b) `hasSegments` ‚Äî which is always true for real transcriptions. The per-job "Speaker Diarization" toggle only sets the server-side `enable_diarization` flag and has NO connection to format. So diarization-off still yields `.srt`. The tooltip ("Transcriptions are saved as .txt (plain text) or .srt/.ass (subtitles ... when diarization is enabled)") and the switch description both claim a diarization‚Üíformat coupling that never existed. The existing srt/ass `<select>` is nested inside `{diarization && (...)}`, reinforcing the false coupling. The server is NOT involved: `/api/transcribe/import`'s docstring explicitly delegates formatting to the client. Do NOT touch `plaintext_export.py`/`subtitle_export.py`/`formatters.py` ‚Äî those belong to the unrelated Notebook/OpenAI-audio subsystems.

**Design decision:** introduce an explicit, always-visible output-format selection for Session imports:
- New per-session setting `outputFormat: 'txt' | 'subtitles' | 'both'` (config key `sessionImport.outputFormat`), shown unconditionally in Import Options.

```ts
export type SessionOutputFormat = 'txt' | 'subtitles' | 'both';

export interface ResolvedOutput {
  outputFilename: string;
  content: string;
}

/**
 * Resolve one or more output files from an explicit, user-chosen format
 * (GH-212). Replaces the implicit hideTimestamps/hasSegments inference for
 * the Session import path. Falls back to .txt when the response carries no
 * usable segments (nothing to build subtitle cues from).
 */
export function resolveTranscriptionOutputs(
  filename: string,
  transcription: TranscriptionResponse,
  options: { outputFormat: SessionOutputFormat; subtitleFormat: 'srt' | 'ass' },
): ResolvedOutput[] {
  const stem = filename.replace(/\.[^.]+$/, '');
  const txt: ResolvedOutput = { outputFilename: `${stem}.txt`, content: renderTxt(transcription) };

  const hasSegments =
    transcription.segments && transcription.segments.some((s) => s.text.trim().length > 0);
  if (options.outputFormat === 'txt' || !hasSegments) return [txt];

  const sub: ResolvedOutput = {
    outputFilename: `${stem}.${options.subtitleFormat}`,
    content:
      options.subtitleFormat === 'ass'
        ? renderAss(transcription, stem)
        : renderSrt(transcription),
  };
  return options.outputFormat === 'both' ? [txt, sub] : [sub];
}
```

**Design decisions:**

import type { JobTrackerProgress } from '../api/types';

export function formatClock(totalSeconds: number): string {
  const s = Math.max(0, Math.floor(totalSeconds));
  const h = Math.floor(s / 3600);
  const m = Math.floor((s % 3600) / 60);
  const sec = s % 60;
  return h > 0
    ? `${h}:${String(m).padStart(2, '0')}:${String(sec).padStart(2, '0')}`
    : `${m}:${String(sec).padStart(2, '0')}`;
}

const PHASE_LABELS: Record<string, string> = {
  loading_model: 'Loading model...',
  transcribing: 'Transcribing',
  diarizing: 'Identifying speakers',
  transcribing_diarizing: 'Transcribing + identifying speakers',
};

/**
 * Human label for an in-flight transcription job (GH-211).
 * `startedAt` is epoch seconds from the server tracker; `nowSeconds` is
 * injected for testability.
 */
export function describeJobProgress(
  progress: JobTrackerProgress | null | undefined,
  startedAt: number | null | undefined,
  nowSeconds: number,
): string {
  const elapsed = startedAt ? Math.max(0, nowSeconds - startedAt) : null;
  const elapsedPart = elapsed !== null ? ` (elapsed ${formatClock(elapsed)})` : '';
  const phase = progress?.phase ?? null;

  if (phase === 'loading_model') return 'Loading model...';

  if ((phase === 'transcribing' || phase === 'transcribing_diarizing') && progress && progress.total > 0) {
    const pct = Math.min(100, Math.round((progress.current / progress.total) * 100));
    let eta = '';
    if (elapsed !== null && elapsed > 5 && progress.current > 0 && progress.current < progress.total) {
      const remaining = ((progress.total - progress.current) * elapsed) / progress.current;
      eta = `, ETA ${formatClock(remaining)}`;
    }
    const verb = PHASE_LABELS[phase];
    return `${verb} ${formatClock(progress.current)} / ${formatClock(progress.total)} (${pct}%${
      elapsed !== null ? `, elapsed ${formatClock(elapsed)}` : ''
    }${eta})`;
  }

  if (phase && PHASE_LABELS[phase]) return `${PHASE_LABELS[phase]}...${elapsedPart}`;

  if (progress && progress.total > 0) {
    return `Processing ${formatClock(progress.current)} / ${formatClock(progress.total)}${elapsedPart}`;
  }
  return `Processing...${elapsedPart}`;
}

```ts
import type { ModelInfo } from './modelRegistry';
import type { OptionMeta } from '../../components/ui/CustomSelect';

export interface ModelCacheEntryLike {
  exists: boolean;
  size?: string;
}

export interface ModelOptionPresentation {
  options: string[];
  optionLabel: Record<string, string>;
  optionDescription: Record<string, string>;
  optionMeta: Record<string, OptionMeta>;
}

/**
 * Build CustomSelect props for a model dropdown: human names, repo-id
 * descriptions, downloaded badges, downloaded-first ordering (GH-213).
 * `tail` holds sentinel options (Disabled/Custom) pinned after the models.
 * A model absent from `cache` has UNKNOWN state - never claim "Not downloaded".
 */
export function buildModelOptionPresentation(
  models: ModelInfo[],
  cache: Record<string, ModelCacheEntryLike | undefined>,
  tail: string[],
): ModelOptionPresentation {
  const downloaded = models.filter((m) => cache[m.id]?.exists === true);
  const rest = models.filter((m) => cache[m.id]?.exists !== true);
  const ordered = [...downloaded, ...rest];

  const optionLabel: Record<string, string> = {};
  const optionDescription: Record<string, string> = {};
  const optionMeta: Record<string, OptionMeta> = {};
  for (const m of ordered) {
    optionLabel[m.id] = m.displayName;
    optionDescription[m.id] = m.id;
    const entry = cache[m.id];
    if (entry?.exists) {
      optionMeta[m.id] = { badge: entry.size ? `Downloaded ${entry.size}` : 'Downloaded' };
    } else if (entry && entry.exists === false) {
      optionMeta[m.id] = { badge: m.approxSize ? `Not downloaded (${m.approxSize})` : 'Not downloaded' };
    } else if (m.approxSize) {
      optionMeta[m.id] = { badge: m.approxSize };
    }
  }
  return { options: [...ordered.map((m) => m.id), ...tail], optionLabel, optionDescription, optionMeta };
}
```

 * Check model cache state while the app container is stopped (GH-213), by
 * mounting the models volume read-only into a throwaway container.
 * Returns {} on any failure - callers must treat missing entries as UNKNOWN,
 * not as "not downloaded".
 */
async function checkModelsCachedOffline(modelIds: string[]): Promise<Record<string, ModelCacheEntry>> {
  try {
    const bin = await runtimeBin();
    const repo = resolveImageRepo(readUseLegacyGpuFromStore(), readRuntimeProfileFromStore());
    const tag = (await readComposeEnvValue('TAG')) || (await listImages())[0]?.tag;
    if (!tag) return {};
    const image = `${repo}:${tag}`;
    const output = await exec(bin, [
      'run', '--rm',
      '-v', `${VOLUME_NAMES.models}:/models:ro`,
      '--entrypoint', '/bin/sh',
      image,
      '-c', 'ls -1 /models 2>/dev/null; echo "---HUB---"; ls -1 /models/hub 2>/dev/null',
    ]);
    const [flatPart, hubPart] = output.split('---HUB---');
    const flatEntries = new Set(flatPart.split(/\r?\n/).map((l) => l.trim()).filter(Boolean));
    const hubEntries = new Set(hubPart?.split(/\r?\n/).map((l) => l.trim()).filter(Boolean) ?? []);
    const result: Record<string, ModelCacheEntry> = {};
    for (const id of modelIds) {
      if (isGgmlFileName(id)) {
        result[id] = { exists: flatEntries.has(path.basename(id.trim())) };
      } else {
        result[id] = { exists: hubEntries.has(hfCacheDirName(id)) };
      }
    }
    return result;
  } catch {
    return {}; // unknown - not "missing"
  }
}
